Under the Hood

How it runs in production

Four stages, one contract: the LLM never returns prose — it returns JSON that matches a fixed schema, so a VA team can act on it without interpretation.

01 · ingest

URL list in

Video URLs from prospecting sheets — one line per creator, thousands per batch.

02 · scrape

Apify actor

Pulls bio, recent captions, follower count, views and engagement rate per profile.

03 · analyze

Claude + strict prompt

A fixed rubric scores niche fit 0–10 against the brand positioning and flags spam patterns.

04 · route

Decision out

≥7.5 goes to outreach, <5 or high spam risk is rejected, the middle gets a human look.

# from source-code/llm_vetting_agent.py — the schema IS the contract
class VettingResult(BaseModel):
    handle: str
    niche: str          # "Clean Beauty", "Minimalist Lifestyle"...
    niche_score: float  # brand fit, 0.0-10.0
    spam_risk: SpamRisk  # LOW / MEDIUM / HIGH
    red_flags: List[str]
    rationale: str      # two sentences, written for an operator

response = client.messages.parse(
    model="claude-sonnet-5",
    system=SYSTEM_PROMPT,      # strict vetting rubric
    output_format=VettingResult,  # schema-validated, never prose
)
